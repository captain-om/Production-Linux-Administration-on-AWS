# Amazon CloudWatch Monitoring

## Objective

The objective of this implementation was to monitor AWS infrastructure and understand the basics of CloudWatch metrics and alarms.

---

## Services Monitored

- Amazon EC2

---

## Metrics Observed

- CPU Utilization
- Network In
- Network Out
- Disk Read Bytes
- Disk Write Bytes
- Status Check

---

## Alarm Configuration

Alarm Name:

HighCPUAlarm

Metric:

CPUUtilization

Statistic:

Average

Threshold:

Greater than 70%

Purpose:

Generate an alert when CPU utilization exceeds the configured threshold.

---

## Monitoring Features

- CloudWatch Metrics
- CloudWatch Dashboard
- CloudWatch Alarm

---

## Result

Successfully monitored the EC2 instance using Amazon CloudWatch and configured a CPU utilization alarm.
